🎯 Goal & Core Finding

Goal: a web dashboard of SpiritVale's player market (vending shops) — current listings, prices, price history, filterable by item and by rolled gear substats.
Core finding (Phase 1, done): The player market is NOT in the game files. It lives in the developer's Google Firebase Firestore backend (198 [FirestoreProperty] fields) and is synced to the client at runtime via the Vending system.

No legitimate direct DB access. Reading Baikun's Firestore directly = against ToS / unauthorized. Workaround = a client capture mod that logs only the market data the player's own client legitimately receives, uploads to our own DB, dashboard reads our DB. Crowdsourced for coverage.

Already extracted from files: 726-item static catalog (data/items.json) + NPC merchant fixed prices (data/merchant_prices.json). This catalog is the reference table live listings join against.

🔄 Section 5 — Rollback Plan

⚠️ If something breaks

1. Game won't start / crashes: remove the mod DLL from BepInEx/plugins (or remove BepInEx winhttp.dll). Game reverts to vanilla. No game files were modified.

2. Mod logs garbage after a patch: stop uploading (delete API key config), re-run Il2CppDumper on the new build, regenerate interop, re-fix hooks by name, rebuild.

3. Bad data in DB: ingest is upsert/append — delete rows by captured_at range or by buildid tag; reseed items from data/items.json.

4. Web app broken: vercel rollback to previous deployment; dashboard is read-only over the DB so no data loss.

5. Abandon entirely: remove mod, drop DB. Static catalog in data/ is unaffected and still usable for an items-only site.
